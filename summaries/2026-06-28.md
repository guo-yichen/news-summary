# 每日摘要 2026-06-28

### 今日要点

1. **OpenAI 发布 GPT-5.6 系列模型**：OpenAI 宣布了 GPT-5.6 系列，包括 Sol、Terra 和 Luna 三个模型，其中 Sol 作为旗舰模型，Terra 为平衡模型，Luna 为快速且经济的模型。这一发布标志着 AI 技术在长视野任务和特定领域应用上的进步，同时受限的预览发布也显示了前沿技术发布与政策监管之间的紧张关系。

2. **GitHub 内部数据分析代理 Qubot**：GitHub 通过其博客详细介绍了如何构建内部数据分析代理 Qubot，这个由 Copilot 驱动的代理使得 GitHub 员工能够用自然语言查询数据仓库中的任何数据模型，并迅速得到答案，这大大提升了数据访问的便捷性和分析效率。

3. **Midjourney Medical 发布超声波 CT 成像技术**：Midjourney Medical 宣布了一种新的全身超声波 CT/超声波系统，这项技术可能革新医疗成像领域，提供非侵入性、无辐射的全身成像，对预防性医疗和疾病早期检测具有重大意义。

4. **Fable 5 改变生产力定义**：Instagram 联合创始人 Mike Krieger 在 Every 的播客中讨论了 Fable 5 模型，这个强大的模型迫使他重新思考生产力、工程和创意代理的定义，Fable 5 的出现预示着软件工程领域的重大变革。

5. **GPT-5.6 系列模型定价与性能**：GPT-5.6 系列模型的定价和性能成为社区讨论的焦点，模型的输入输出定价以及在不同任务上的表现，尤其是 Sol 模型在编程、网络和长期工作任务上的能力，引起了业界的广泛关注。

#### GitHub Blog (AI)
- **How we built an internal data analytics agent**  
  「核心观点」GitHub 通过构建内部数据分析代理 Qubot，实现了数据访问的自助服务。  
  主讲人 Weijie Tan 和 Tobias Tschuemperlin 介绍了 Qubot 的三个主要组件：用户界面、上下文层和查询引擎。Qubot 允许 GitHub 员工用自然语言询问数据模型问题，并迅速得到答案。上下文层通过产品团队、数据和分析团队以及拥有数据集的团队贡献的知识来构建，而上下文代理则通过标准化模板或参考仓库来集中贡献。评估框架在变更发布前对上下文层或代理配置进行评估，确保准确性和性能。Qubot 连接到 GitHub 的两个查询引擎 Kusto 和 Trino，根据问题的复杂性自动切换。  
  “Qubot has zero cost maintenance and helps teams ramp up quickly on datasets they may be unfamiliar with.”  
  [阅读全文](https://github.blog/ai-and-ml/github-copilot/how-we-built-an-internal-data-analytics-agent/)
