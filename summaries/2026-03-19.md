**1. Anthropic 军事合作争议持续发酵**

本周 Anthropic 连续发布两份声明，分别来自 CEO Dario Amodei 以及针对美国战争部长 Pete Hegseth 评论的回应，涉及与美国国防部门的合作讨论。这是 AI 公司与军事机构关系这一长期敏感议题的最新进展，对整个行业的监管走向和 AI 伦理边界有重要示范意义。两份声明相隔一天发出，说明局面仍在快速演变。[声明一](https://www.anthropic.com/news/statement-department-of-war) | [声明二](https://www.anthropic.com/news/statement-comments-secretary-war)

**2. Anthropic 收购 Vercept，重押 computer use 方向**

Anthropic 宣布收购 Vercept，明确目的是推进 Claude 的计算机操控能力（computer use）。这意味着 Anthropic 正在将 agent 能力从"对话"向"执行"层面深推——不再只是生成文字，而是真正控制软件界面完成任务。此举与 GitHub、Cursor 等工具生态同步向 agentic 工作流演进的趋势高度吻合。[链接](https://www.anthropic.com/news/acquires-vercept)

**3. OpenAI 发布 GPT-5.4 mini 和 nano：效率端的军备竞赛**

GPT-5.4 的两个轻量版本正式上线，针对编程、工具调用、多模态推理和高并发 API 场景优化。这延续了大厂"旗舰模型发布后快速推出小模型"的节奏，对开发者而言意味着更低成本构建 sub-agent 工作流的可能。与此同时，Windsurf 已宣布 GPT-5.4 完整版上线，MiniMax 2.7 也以 1/3 成本实现 SOTA 级别性能，效率端竞争白热化。[链接](https://openai.com/index/introducing-gpt-5-4-mini-and-nano)

**4. The Pragmatic Engineer 深度报道：AI agent 正在拖慢研发速度**

这篇文章直指一个反直觉现象：随着越来越多工程师日常使用 AI agent，代码质量下降、故障增加、交付速度反而变慢。这是目前关于 AI 辅助编程最值得警惕的实证观察之一，对正在或准备大规模铺开 AI 工具的工程团队有直接参考价值。[链接](https://newsletter.pragmaticengineer.com/p/are-ai-agents-actually-slowing-us)

**5. Cursor 发布自总结训练论文，突破 context window 限制**

Cursor 团队公布了一项技术：通过将"自我总结"纳入 Composer 的训练过程，可以从远超模型最大 context window 的轨迹中获取训练信号。这是解决长任务 agent 训练瓶颈的实质性技术进展，而非工程 trick，值得 AI 工程和研究从业者深读。[链接](https://cursor.com/blog/self-summarization)

### Claude Code Changelog

Claude Code 近期连续发布多个版本（v2.1.75–v2.1.79），更新密集，以下是跨版本的关键变化：

**功能层面**
- **v2.1.79** 新增 `--console` 认证标志，并引入 [VSCode] `/remote-control` 功能，可将本地 session 桥接至 claude.ai/code，从浏览器或手机继续工作。[链接](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#2179)
- **v2.1.77** 将 Claude Opus 4.6 的默认最大输出 token 提升至 64k，上限扩展至 128k；Opus 4.6 的 1M context window 现对 Max/Team/Enterprise 计划默认开放（此前需额外购买）。[链接](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#2177)
- **v2.1.76** 新增 MCP elicitation 支持（server 可在任务中途请求结构化输入）、`/effort` 命令调节模型推理强度、`PostCompact` hook 等。[链接](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#2176)

**性能与稳定性**
- v2.1.77 startup 性能提升约 60ms（macOS），`--resume` 速度提升最高 45%，峰值内存降低约 100–150MB。
- v2.1.79 startup 内存使用减少约 18MB。
- v2.1.78 新增 `StopFailure` hook，改进非流式 API 降级逻辑，防止 session 无限挂起。

**安全修复**
- v2.1.78 修复 `PreToolUse` hook 返回 `"allow"` 绕过 `deny` 权限规则的安全漏洞（包括企业托管设置）；修复 sandbox 缺少依赖时被静默禁用的问题。[链接](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#2178)
