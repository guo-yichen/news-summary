## 今日要点

**1. OpenAI 发布 GPT-5.4 mini 和 GPT-5.4 nano，价格创新低**

GPT-5.4-nano 的输出定价仅为每百万 token $1.25，甚至比 Google Gemini 3.1 Flash-Lite 还便宜。Simon Willison 实测用 GPT-5.4-nano 描述一张照片只花了不到 0.1 美分，处理其 76,000 张照片库的总成本仅约 $52。这意味着视觉 AI 应用的边际成本正在趋近于零，大规模图像理解、自动标注等场景将迎来爆发式普及。这一定价压力对 Claude 和 Gemini 系列中端模型构成直接竞争。

**2. Anthropic 与"战争部"的持续博弈：AI 军事应用的边界之争**

Anthropic 在 2026 年 2 月底至 3 月初连续发布三篇声明，内容围绕与美国"战争部"（Department of War）的谈判。从声明标题的演变——从"讨论"到"Secretary of War Pete Hegseth 的评论"再到"事态进展"——可以看出这是一场持续升温的公开张力。这是 AI 头部公司与政府军事机构之间价值观冲突公开化的标志性事件，其走向将深刻影响行业内其他公司的政策制定。

**3. Claude Code 大规模更新，MCP elicitation 和 worktree 功能成亮点**

v2.1.74 至 v2.1.76 三个版本密集落地，新增 MCP elicitation（支持 AI 任务中途通过表单向用户请求结构化输入）、git sparse-checkout 支持大型 monorepo、`/effort` 命令调节模型推理深度，以及 Opus 4.6 在 Max/Team/Enterprise 计划中默认开放 1M 上下文窗口。这些更新显示 Claude Code 正在从"聪明的代码补全"向"可与用户深度交互的生产级 agent"演进。

**4. Uber 内部 AI 工具体系曝光：Minion、Shepherd、uReview 的工程实践**

The Pragmatic Engineer 深度报道了 Uber 如何构建一套完整的内部 agentic AI 工具链。其中 Minion 负责代码生成，Shepherd 管理 agent 工作流，uReview 自动化代码审查。文章还揭示了企业落地 AI 工具的新挑战：平台投入需求激增、token 成本快速膨胀。这是目前最具参考价值的大厂 AI 工程化实践案例之一。

**5. Simon Willison 系统整理 Subagent 工程模式，附 Claude Code 真实案例**

Willison 发布了一篇关于 subagent 架构的深度指南，用 Claude Code 的 Explore subagent 作为具体案例，揭示了为什么 subagent 是解决 context 限制的关键模式——它让 agent 可以派出"新鲜的自己"完成子任务，而不消耗主 context。文章还介绍了并行 subagent 和专家型 subagent 的用法，并汇总了 OpenAI Codex、Gemini CLI、Cursor 等平台的官方 subagent 文档链接，是当前 agentic engineering 最值得精读的实操资料。

### Claude Code Changelog

三个版本密集发布，核心亮点如下：

**v2.1.76** 是功能最丰富的版本。**MCP elicitation** 是最重磅的新特性——MCP server 现在可以在任务执行过程中通过对话框（表单或浏览器 URL）向用户请求结构化输入，使 AI agent 可以"按需暂停并询问"，大幅提升复杂任务的可控性。此外新增 `PostCompact` hook、`/effort` 命令调节推理力度、`worktree.sparsePaths` 支持大型 monorepo 稀疏检出。修复了十余个 bug，包括 context compaction 后工具 schema 丢失、MCP OAuth 认证死锁等生产级问题。[链接](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#2176)

**v2.1.75** 的核心亮点是 **Opus 4.6 默认开放 1M 上下文**（无需额外付费，适用于 Max/Team/Enterprise）。另新增 `/color` 命令为 session 设置提示栏颜色、记忆文件最后修改时间戳（帮助 Claude 判断记忆新鲜度）。[链接](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#2175)

**v2.1.74** 以稳定性修复为主，修复了内存泄漏（streaming API 响应缓冲区未释放，导致 Node.js 路径下 RSS 无限增长），并修复了 managed policy `ask` 规则被用户 `allow` 规则绕过的安全问题。[链接](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md#2174)
