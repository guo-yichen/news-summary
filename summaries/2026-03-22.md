### Simon Willison's Blog

**用 LLM 给 Hacker News 用户做画像**

用户公开评论数据 + LLM 推理 = 惊人精准的人物侧写，这是 Simon Willison 这篇文章最重要的结论。

Simon Willison 是独立开发者、Django 联合创始人、Datasette 作者，Python 软件基金会董事会成员，长期在 simonwillison.net 运营活跃的技术博客。

这个实验的操作极简：调用 Algolia Hacker News API 抓取任意用户最近 1000 条评论，粘贴进 Claude Opus 4.6，prompt 只需写「profile this user」。Claude 生成的画像精准到令人不安——职业身份、技术立场、写作风格、人生观、重复出现的关键词，甚至真实姓名都能被推断出来。

反直觉的洞见有三：第一，「自愿公开的数据」并不等于「不可被整合分析」，聚合效应会产生远超单条评论的信息量；第二，Willison 特别说明自己用了 Claude 的「隐身模式」以防模型认出自己然后「谄媚」，这个操作设计本身就很值得借鉴；第三，他主要用这个工具来「筛查是否在和一个长期论战的坏信仰账号争论」——一个非常务实的防御性用途。

原文中最有力的一句话来自 Claude 生成的画像：*「A deeply experienced, independently-minded developer who's genuinely excited about AI coding tools, fights hard against both uncritical hype and reflexive dismissal, and is trying to professionalize how the industry uses these tools — while also worrying loudly about the security implications almost nobody else takes seriously enough.」*

[原文链接](https://simonwillison.net/2026/Mar/21/profiling-hacker-news-users/#atom-everything)

**Coding agents 与 Git 的协作模式**

这篇文章的核心结论是：Git 的高阶功能（bisect、历史重写、冲突处理）的学习曲线曾经是开发者的主要障碍，现在这个障碍已经被 coding agents 消除了——你只需要知道「这个功能存在」，不需要记住怎么用。

- **关键要点**：
  - 「Review changes made today」是极佳的 agent 会话开启方式，让 agent 跑 `git log` 加载上下文，比手动解释背景高效得多
  - `git bisect` 是「最强但最少被使用的调试工具」，难点在于编写测试条件的样板代码，agents 可以完全接管这部分，将其升级为日常可用工具
  - 「Sort out this git mess for me」这种模糊 prompt 实际有效，agents 能处理 merge conflict 的复杂推理
  - 「从旧仓库提取代码到新仓库同时保留提交历史」这类过去几乎没人愿意做的操作，现在用自然语言即可完成
  - Willison 承认自己已接受 frontier model 生成的 commit message 质量，「often even better than what I would have produced myself」
- **值不值得深读**：强烈推荐，是少有的将 agentic workflow 与具体工程实践结合的系统性指南。

[原文链接](https://simonwillison.net/guides/agentic-engineering-patterns/using-git-with-coding-agents/#atom-everything)

**用 Claude Artifacts 制作排序算法动画**

- **核心论点**：Willison 展示了一个典型的「vibe coding」案例：用自然语言 prompt 在手机上通过 Claude Artifacts 构建排序算法可视化工具，并让 Claude 直接从 GitHub 克隆 CPython 仓库查阅 Timsort 源码实现。
- **关键要点**：
