## 今日要点

**1. Andrej Karpathy：代码 agent 已经彻底改变软件工程师的工作方式**

No Priors 播客最新一期专访 Karpathy，他描述了一个极为震撼的转变：从 2025 年 12 月起，他几乎不再亲手写一行代码，工作方式从"自己写代码"变成了"向 agent 传达意志"。他用"AI psychosis（AI 精神病状态）"来形容自己持续处于探索 agent 极限的亢奋状态。更惊人的是，他用自己开发的 AutoResearch 工具让 AI 自主跑了 70 多次实验，把一个用 Rust 写的象棋引擎从"专家级"提升到了全球第 311 名、ELO 2718 的大师水平。这不只是个人体验——他认为这是整个行业的拐点，普通人还没意识到这件事有多剧烈。

**2. Cursor 发布 Composer 2：押注"快而便宜"而非"完全委托"**

Cursor 的 Composer 2 代表了与 Claude Code 和 OpenAI Codex 截然不同的产品哲学。后两者优化"委托式"工作流（描述问题，等 AI 解决），而 Cursor 坚持让开发者留在循环中、实时管理 agent。Composer 2 的价格仅为 Opus 4.6 的 1/10，速度超过最快的 Opus 模型，基准测试达到 frontier 水准——但测评者指出它只在 Cursor 特定工作流内才能充分发挥优势。这场关于"AI 编程应该是委托还是协作"的路线之争，正在快速分化用户群体。

**3. Uber 自动驾驶布局存在严重数字泡沫，或难敌 Waymo**

The Leverage 对 Uber CFO 的"2029 年全球最大 AV 车队"豪言做了独立数据核查，结论是：即使所有合作伙伴都按时交付，Uber 到 2029 年的 AV 总量也不过约 2.8 万辆，而 Waymo 届时预计已部署超过 5 万辆。更关键的问题是：Uber 的核心合作伙伴 Rivian 在 SEC 文件中还只是"打算开发" Level 4 能力，技术准备严重滞后。这篇分析提供了一个重要的反向视角：在 AV 领域，平台分发能力并不等同于技术领先。

**4. Sebastian Raschka 发布 LLM 注意力机制全景视觉指南**

Sebastian Raschka（机器学习研究者，著有《Build a Large Language Model from Scratch》）发布了一篇系统梳理现代 LLM 中各种 attention 变体的深度文章，配套他维护的 45 个条目的 LLM 架构图鉴（含可视化 model card）。文章涵盖从 MHA（多头注意力）到 GQA、Sliding Window Attention 等各种变体，并追溯了 attention 机制从 RNN 时代到 Transformer 时代的演变历史。对于想系统理解现代 LLM 架构的工程师和研究者，这是目前少有的兼顾广度与深度的参考资料。

**5. Zara Zhang 一针见血：AI 让人更忙，而非更闲**

Zara Zhang（投资人）的一条推文精准描述了 AI power user 的真实体验：大家以为 AI 会带来 10 倍生产力从而可以早点收工，实际上是 10 倍生产力带来了 20 倍的待办事项——因为可能性本身在膨胀。这个观察与 Karpathy 描述的"AI psychosis"相互印证，揭示了 AI 时代一个深层的结构性张力：工具越强，人反而越难停下来。

### Ahead of AI (Sebastian Raschka)

**核心论点**：这是一篇系统梳理现代 LLM 中所有主流 attention 变体的视觉参考指南，兼具学习价值和工程查阅价值。

**关键要点**
- Raschka 同步维护了一个包含 45 个条目的 LLM 架构图鉴，每个架构都有可视化 model card，并计划持续更新
- 文章从 attention 的历史起源讲起：它最初是为了解决 RNN 编码器-解码器架构中"隐藏状态无法存储足够上下文"的瓶颈而发明的
- MHA（多头注意力）的核心机制：多组 Q/K/V 并行计算，各自学到不同的 token 关系，最后合并输出
- Transformer 的关键创新是把 attention 从 RNN 的"辅助机制"升级为"主处理机制"，彻底去掉了循环结构
- 除 MHA 外，文章还涵盖 GQA（分组查询注意力）、Sliding Window Attention 等现代变体，并标注了哪些架构在使用哪种变体
- 配套提供了海报版本（推荐 Medium 及以上尺寸，小尺寸文字会过小）

**值不值得深读**：非常值得，是目前少有的将 attention 机制历史、原理、变体和实际架构应用串联起来的系统性参考。

[原文链接](https://magazine.sebastianraschka.com/p/visual-attention-variants)

### Chips and Cheese

**核心论点**：这篇文章通过实测数据量化了不同平台上主板芯片组 vs CPU 直连 PCIe 的延迟差异，结论是经过芯片组的 PCIe 连接会带来 300-900ns 不等的额外延迟。

**关键要点**
- 使用 Vulkan 基准测试，测量 GPU 访问 host memory 时经过不同路径的延迟
- AMD AM5 平台（Zen 5）：CPU 直连约 650ns，经过一个 PROM21 芯片组约增加 570ns，经过两个（X670E）则增加约 921ns
- Intel Arrow Lake（Z890）：CPU 直连约 785ns，经过 PCH 约增加 550ns，与 AM5 单芯片组接近
- Intel Skylake（Z170）：CPU 直连仅 536ns，经过 PCH 仅增加 338ns，表现优于当代平台
- GPU 缓存命中带宽在芯片组连接时会受到约束（在 AM5 上降至约 25 GB/s）

**值不值得深读**：对硬件性能调优感兴趣的工程师值得看，普通 AI 从业者可跳过。

[原文链接](https://chipsandcheese.com/p/microbenchmarking-chipsets-for-giggles)

### Context Window

**核心论点**：Cursor 的 Composer 2 以极低价格和极快速度进入 frontier 性能区间，但它服务的是"开发者留在循环中"这一特定工作流，而非与 Claude Code / Codex 的"完全委托"路线正面竞争。

**关键要点**
- Composer 2 标准版定价为 Opus 4.6 的 1/10，GPT-5.4 的 1/5；"fast"版速度超过最快的 Opus 模型
- 基准测试显示 Composer 2 达到 frontier 水准，但测评者指出这只在 Cursor 特定工作流中成立
- Claude Code 和 Codex 的产品哲学是"委托"：你描述问题，AI 去解决，你只需审查结果

- 这场路线分歧正在形成两类截然不同的用户群体和使用习惯

**值不值得深读**：值得，对理解 AI 编程工具的竞争格局和产品哲学分化有直接价值。（注：全文为付费内容）

### No Priors（播客）

**Andrej Karpathy on Code Agents, AutoResearch, and the Loopy Era of AI**

核心观点是：**以 2025 年 12 月为分水岭，软件工程师的工作方式发生了本质性的、大多数人还没意识到的剧变——写代码这个动作本身正在消失，取而代之的是"向 agent 表达意志"。**

Andrej Karpathy（前 OpenAI 研究负责人，前 Tesla AI 总监，现独立研究者）在 No Priors 播客中分享了他过去几个月的第一手体验。他用"AI psychosis"来描述自己持续处于探索 agent 边界的亢奋状态，并透露从 2025 年 12 月起几乎不再亲手写代码。

三个最有价值的洞见：

**第一**，agent 的使用方式正在快速层叠升级——从单个 session，到多个并发 session，到给 agent 写指令，再到对指令本身进行优化——每一层都是新的技能点，也是新的"卡脖子"瓶颈，他称之为"一切皆 skill issue"。

**第二**，他用自己的 AutoResearch 工具自主运行了 70 多次实验，最终将一个 Rust 象棋引擎的 ELO 从"专家级"推到了 2718，跻身全球第 311 名——这是 AI 自主科研能力的一次具体展示。

**第三**，他用 OpenClaw 通过发短信来控制家中的 Sonos 音响、灯光、暖通空调和安防系统，agent 会自动扫描局域网、找到硬件并接入——完全不需要写代码。

> "Code's not even the right verb anymore. I have to express my will to my agents for 16 hours a day. Manifest."
