## 今日要点

**1. OpenAI 发布 GPT-5.3 Instant，同时与美国国防部签署协议**

OpenAI 在同一周内完成了两个高度敏感的动作：发布主打日常对话流畅度的 GPT-5.3 Instant，以及正式宣布与美国"战争部"（Department of War）达成合作协议。后者尤为值得关注——这标志着 OpenAI 从"安全 AI"叙事向军事 AI 应用迈出了实质性一步，是 AI 产业政治化的重要信号。两篇发布同时上线，但具体合同条款和技术范围尚不清楚，值得持续追踪。

**2. "Harness Engineering 是否真的有价值"成为 AI 工程界最核心争论**

Latent Space 的深度分析把这个问题摆上了台面：当模型越来越强，围绕模型搭建的 agent 框架（harness）还有多少独立价值？Claude Code 团队明确表示"我们的 harness 越做越薄、越做越简单，所有秘密都在模型里"；而 OpenAI 的 Noam Brown 直接说 reasoning model 出现后，复杂 scaffold 大多会被模型能力取代。但另一边，Cursor 估值已达 $50B，Big Harness 阵营也拿出了数据反驳。这场争论直接决定了未来 AI 工程师的核心技能方向。

**3. Andrej Karpathy 自述"AI 精神病"状态：4个月没手写一行代码，每天16小时与 agent 协作**

根据 Twitter 上流传的播客内容摘要，Karpathy 表示自己从去年 12 月起就完全停止手写代码，转为全天候与多个并行 agent 协作。这不是噱头——Karpathy 是全球最被信任的 AI 实践者之一，他的工作流转变往往预示着整个工程师群体的未来走向，值得认真对待。

**4. Simon Willison 演示用 LLM 对 Hacker News 用户进行画像分析，效果"令人不安地有效"**

只需要用户最近 1000 条公开评论，Claude Opus 4.6 就能生成极为准确的个人画像，涵盖职业身份、技术立场、辩论风格、个人兴趣。Willison 以自己为实验对象，结果高度吻合。这项实验揭示了一个重要的隐私问题：即便是完全公开的数据，经过 LLM 聚合分析后也会产生强烈的"被监视感"，对 AI 伦理和数据隐私的讨论具有直接参考价值。

**5. Simon Willison 系统整理"coding agent 与 Git 协作"的最佳实践**

这是一篇高度实操性的指南，核心论点是：coding agent 对 Git 的掌握程度已经远超大多数开发者的预期，包括 `git bisect`、历史重写、跨仓库代码迁移等高级操作。文章中有一个反直觉洞见：开发者不需要记住 Git 命令，只需要知道"有这个能力存在"，让 agent 去执行。这直接改变了开发者需要掌握的知识结构。

### Simon Willison's Blog

**用 LLM 对 Hacker News 用户进行画像分析**

**核心论点**：公开的社交媒体数据经过 LLM 聚合分析后，可以生成令人惊讶地准确、详细的个人画像，这在技术上轻而易举，在伦理上值得警惕。

**关键要点**：
- 只需调用 Algolia HN API 获取用户最近 1000 条评论，然后用 Claude Opus 4.6 提示 "profile this user"，即可生成多维度画像
- 画像内容涵盖：职业身份、核心技术立场、工作风格、辩论偏好、个人兴趣、甚至预测偏误
- Willison 以自己为实验对象，Claude 的分析高度准确，包括他"喜欢在 BART 上用 iPhone 写代码"这类细节
- API 支持 CORS，意味着任何网页上的 JavaScript 都可以直接调用，门槛极低
- Willison 坦承："It's a little creepy to be able to derive this much information about someone so easily"
- 他实际用这个工具的目的是：在深度争论前，先确认对方是否有恶意争论历史

**值不值得深读**：值得，对 AI 伦理、数据隐私、LLM 能力边界都有直接启示。

[原文链接](https://simonwillison.net/2026/Mar/21/profiling-hacker-news-users/#atom-everything)

**Coding Agent 与 Git 协作指南**

**核心论点**：coding agent 对 Git 的掌握已经高度成熟，开发者应该主动把更多 Git 操作——包括历史重写、bisect 调试、跨仓库迁移——委托给 agent 执行，而不是死记命令。

**关键要点**：
- "Review changes made today" 是启动新 agent 会话的最佳方式，让 agent 通过 `git log` 快速载入上下文
- `git bisect` 是最被低估的调试工具，学习曲线高但 agent 可以完全接管；Willison 建议把它从"偶尔用"升级为"随时用"
- "Sort out this git mess for me" 这类自然语言指令对 agent 完全有效，merge conflict 可以直接交给 agent 解决
- Agent 的 commit message 质量通常足够好，Willison 已经放弃坚持自己写
- 可以指示 agent 从大型仓库中提取特定模块到新仓库，同时保留关键 commit 历史和原始时间戳
- Git reflog 可以帮助找回未正式 commit 的代码，agent 可以搜索和恢复

**值不值得深读**：非常值得，是 agentic engineering 实操中最有直接价值的指南之一。

[原文链接](https://simonwillison.net/guides/agentic-engineering-patterns/using-git-with-coding-agents/#atom-everything)

**[AINews] Harness Engineering 是否真实存在？**

**核心论点**：AI agent 工程领域最核心的架构之争正在上演——"Big Model"阵营认为模型越强 harness 越应该薄，"Big Harness"阵营则认为 context 和 workflow 工程才是真正的护城河，而 $50B 估值的 Cursor 是后者最有力的市场答案。

**关键要点**：
- Claude Code 团队 Boris Cherny 明确表态："所有秘密都在模型里，我们的 harness 是能做到的最薄的包装，而且还在越做越薄"
- OpenAI Noam Brown 认为：reasoning model 出现后，之前为 GPT-4o 设计的复杂 agentic scaffold 大多变得多余甚至有害；未来 model router 也会被单一统一模型取代
- Scale AI 的 SWE-Atlas 数据显示：Claude Opus 4.6 在 Claude Code 里比通用 SWE-Agent 高 2.5 分，但 GPT-5.2 结果相反，说明 harness 的影响在误差范围内
- METR 评估发现 Claude Code 和 Codex 在 agent 基准上并不优于简单 scaffold
- 另一方：LlamaIndex 创始人 Jerry Liu 坚持认为"context 和 workflow 工程是从 AI 获取价值的最大障碍"
- 有研究显示仅优化 harness（Pi）就让 15 个 LLM 的编程性能显著提升
- Gemini 3.1 Flash-Lite 泄露信息、GPT-5.4 传闻（1M token context window + "极端推理模式"，可"思考数小时"）等前沿模型动态也在本期中被整理

**Every Agent Needs a Box — Aaron Levie, Box CEO**

**核心论点**：企业存储的核心价值正在从"帮人管文件"转变为"给 agent 提供安全可信的数据沙箱"，Box CEO Aaron Levie 认为 agent 时代的到来是企业内容平台的最大历史机遇。

**关键要点**：
- Levie 的核心洞见：企业文件里存储着大量"沉睡知识"（合同、研究材料、路线图），人类只在主动工作时接触它们；但 agent 可以随时主动调用这些数据产生价值
- 他区分了两类 agent：一类"作为你"运行（访问你有权限的数据）；另一类完全自主运行（像同事一样拥有独立环境和工具）
- "Every agent needs a box"——agent 需要一个有权限边界的受信数据环境，而不是裸连所有企业数据
- 节目中讨论了 agent governance（身份认证、权限控制）是目前被严重低估的基础设施问题
- Aaron 指出：现在的变化不是 agent 适应人类工作方式，而是人类在主动改变工作方式去配合 agent
- Box 的 ARR 已超过 $1.1B，利润率 28%，在"AI 正在杀死 SaaS"的叙事中逆势增长

**值不值得深读**：值得，特别是对企业 AI 基础设施、agent 权限管理感兴趣的读者。

[原文链接](https://www.latent.space/p/box)
