## 今日要点

**1. ByteDance DeerFlow 2.0 登顶 GitHub Trending**
字节跳动开源的 DeerFlow 2.0 在 2026 年 2 月 28 日发布后荣登 GitHub Trending 第一位。这次是完全重写，与 v1 不共享任何代码，定位从「深度研究框架」升级为「超级 agent 框架」，支持子 agent 编排、长期记忆、沙盒执行、MCP 扩展以及 Telegram/Slack/飞书等 IM 渠道接入。这标志着开源 agent 框架正在从单一研究用途向通用任务自动化演进，值得重点关注。

**2. OpenAI 宣布不再使用 SWE-bench Verified 评估**
OpenAI 发布文章解释为何放弃 SWE-bench Verified 作为评测基准。这一举动在 AI 评测领域意义重大——SWE-bench 此前几乎是衡量代码 agent 能力的行业标准，OpenAI 的主动弃用暗示该基准可能已被「过拟合」或不再能真实反映模型能力，可能引发整个行业对 benchmark 选择的重新审视。

**3. Dan Shipper 提出 2026 工程团队新模型：「海盗 + 架构师」二人组**
Every 创始人 Dan Shipper 提出一个具体的工程团队结构：只需 2 人，一名「海盗」负责用 vibe coding 极速交付产品功能，一名「架构师」负责将探索出的产品表面固化为可扩展的系统。这一模型非常具体且有争议性，反映出 AI 辅助编程正在根本改变小团队的组织逻辑。

**4. Simon Willison 用 Claude skills 驱动 Starlette 1.0 代码生成**
Simon Willison 详细记录了一个实操案例：先让 Claude 自动克隆 Starlette 1.0 仓库并生成 skill 文档，再用该 skill 直接生成可运行的 Starlette 1.0 应用。这个案例揭示了一个重要工作流——通过 skill 注入最新文档，可以让 LLM 生成训练数据截止日期之后发布的新框架代码，解决了模型知识时效性问题。

**5. Karpathy 的 Autoresearch 工具让振动编程（vibe-coded）Rust 棋局引擎跑到国际象棋 ELO 2718**
@Deedy 分享：用 Karpathy 的 Autoresearch 工具对一个 vibe-coded Rust 国际象棋引擎自动跑了 70+ 次实验并持续爬坡优化，最终达到 ELO 2718（全球第 311 名引擎）。这是 AI 自动化科学实验（AutoResearch）能力的一个非常具体的实测案例，说明 agent 驱动的迭代优化已经可以产生真正有竞争力的结果。

### Simon Willison's Blog

**文章：Experimenting with Starlette 1.0 with Claude skills**

- **核心论点**：通过让 Claude 自动生成 Starlette 1.0 的 skill 文档，解决 LLM 训练数据无法覆盖新版本框架的问题，并成功生成可运行的 Starlette 1.0 应用
- **关键要点**：
  - Starlette 1.0 正式发布，这是自 2018 年以来的重大里程碑；FastAPI 的底层框架，实际使用量远超品牌知名度
  - 1.0 最大变化是用基于 async context manager 的 lifespan 机制替换了旧的 on_startup/on_shutdown 参数
  - 工作流：让 Claude 克隆 GitHub 仓库 → 自动生成 skill markdown 文档 → 点击「Copy to your skills」注入到 Claude Chat → 后续对话即可生成符合 1.0 规范的代码
  - Claude 在生成任务管理 demo 应用后，还主动用 TestClient 自行测试了 API 端点，展现了 coding agent 的完整工作流
  - 原文引用：*「For all of the buzz about Claude Code, it's easy to overlook that Claude itself counts as a coding agent now, fully able to both write and then test the code that it is writing.」*
- **值不值得深读**：非常值得，提供了一个处理 LLM 知识时效问题的可复用工作流
- 🔗 [原文链接](https://simonwillison.net/2026/Mar/22/starlette/#atom-everything)
