## 今日要点

**1. Cursor Composer 2 基于 Kimi K2.5 构建，中国模型悄然进入西方主流开发工具**

Kimi.ai 官方确认，Cursor 新发布的 Composer 2 是在其 Kimi-k2.5 模型基础上，经过 Cursor 团队持续预训练和高算力 RL 训练后构建的，推理和推断由 FireworksAI 平台提供支持。这是一个重要信号：中国 AI 模型正以"授权商业合作伙伴"的形式悄然成为西方顶级开发工具的核心底座，而非仅仅作为竞争对手存在。Dan Shipper 实测 Composer 2 赢过 GPT-5.4，Ryo Lu（Windsurf）也在 Twitter 上为 Composer 2 背书。

**2. OpenAI 发布 GPT-5.4 Thinking System Card 及 CoT 可控性研究**

OpenAI 同日发布了两项重要内容：GPT-5.4 Thinking 的 System Card，以及一项关于 reasoning model 的 CoT 可控性研究。研究核心发现是：reasoning model 很难控制自己的思维链，而这恰恰是一件好事——因为这说明模型的 chain of thought 具备真实的 monitorability（可监控性），是 AI 安全的重要保障，不易被训练成表面合规而内部欺骗。

**3. 每个主要 AI 实验室都在收购开发者工具——行业整合加速**

Latent Space Newsletter 报道了一个值得警觉的趋势：OpenAI 收购 Astral、Anthropic 收购 Bun、Google DeepMind 收购 Antigravity 团队。这表明各大实验室已意识到，仅靠模型能力无法建立护城河，必须垂直整合开发者工具链，将开发者生态牢牢锁定在自己平台上。

**4. /dev/agents 正式更名为 Dreamer，提出"个人 Agent OS"愿景**

Latent Space Newsletter 披露，此前低调运作的 /dev/agents 正式以 Dreamer 品牌出现，目标是构建一个个人级别的 Agent 操作系统。这一愿景极其雄心勃勃：让 AI agent 不只是工具，而是用户个人计算体验的核心操作层。项目同步宣布 $10,000 工具开发奖励，并为 Latent Space 订阅者提供特殊访问权限。

**5. Simon Willison 在 Pragmatic Summit 的 agentic 工程演讲：测试已经免费，不写测试是你自己的选择**

Simon Willison（Datasette 创始人）在演讲中提出了一系列高密度实操洞见：coding agent 时代 TDD 变成零成本选项、"conformance-driven development"的新范式、以及 prompt injection 的"致命三元组"安全风险框架。最有力的一句话是：「Tests are effectively free. I think tests are no longer even remotely optional.」值得所有使用 coding agent 的工程师认真阅读。

**Reasoning models struggle to control their chains of thought, and that's good**

- **核心论点**：OpenAI 研究发现，reasoning model 很难被训练成"按需控制"自己的思维链，而这一特性反而强化了 AI 安全中的 monitorability（可监控性）保障。
- **关键要点**：
  - OpenAI 引入了"CoT-Control"这一评估框架，用于测量模型对自身 chain of thought 的可控程度
  - 研究发现，当尝试训练模型隐藏或伪造推理过程时，模型的行为难以稳定控制
  - 这意味着 CoT 作为安全监控手段具有相当的鲁棒性，不容易被训练成"表面顺从、内部欺骗"
  - 这一发现对 AI alignment 和 interpretability 研究具有实质性意义
- **值不值得深读**：非常值得，这是目前 reasoning model 安全性研究中少有的实证性发现。
- [原文链接](https://openai.com/index/reasoning-models-chain-of-thought-controllability)

核心观点是：Claude 能够读取并反编译 1985 年的 39,731 字节 Turbo Pascal 二进制文件，生成带注释的汇编代码，并构建出一个可交互的展示 artifact。Simon 展示了完整的提示词序列：先让 Claude 读一篇背景文章，再让它在线找到二进制文件，最后构建一个嵌入该二进制的无 React 交互 artifact。这是一个展示 LLM 在逆向工程和计算机历史文档化方面能力的有趣案例。

**My fireside chat about agentic engineering at the Pragmatic Summit**

这是目前关于 coding agent 实战使用最密度最高的一篇一手经验总结，核心观点是：**测试现在已经免费，不写测试是工程师自己的主动放弃**。

Simon Willison 是 Datasette 创始人、Django 联合创始人，连续多年深度使用 coding agent。他在演讲中提炼了以下几个反直觉洞见：

- **TDD 变了性质**：他一生厌恶测试先行，但让 agent 做 TDD 完全不同——「I don't care if the agent spins around for a few minutes wasting its time on a test that doesn't work.」测试写作的心理成本归零。

- **"Conformance-driven development"新范式**：通过让 Claude 先为 Go/Node.js/Django 等多个已有实现写出统一测试套件，再用该测试套件驱动新实现，相当于从六个实现逆向工程出一个标准，再实现这个标准。

- **prompt injection 的"致命三元组"**：私有数据访问权 + 恶意指令暴露 + 外泄向量，三者同时存在时才构成高危场景。他坦承自己明知风险仍在 Mac 上跑 `--dangerously-skip-permissions`，因为太好用了。

- **代码质量是选择题**：「If the agent spits out 2,000 lines of bad code and you choose to ignore it, that's on you.」坏代码是工程师自己的选择，而非 agent 的必然输出。

- **开源面临挑战**：「Projects are flooded with junk contributions to the point that people are trying to convince GitHub to disable pull requests.」

- [原文链接](https://simonwillison.net/2026/Mar/14/pragmatic-summit/#atom-everything)

### Data Driven NYC

**Benedict Evans: OpenAI's Moat Problem & the Future of Software**

Benedict Evans（独立科技分析师，前 a16z 合伙人）与 Matt Turck 的深度对话，核心论点是：**LLM 没有赢者通吃效应，这从根本上颠覆了 OpenAI 的商业逻辑。**

几个最值得关注的论断：

- **OpenAI 的根本困境**：「There is no winner takes all effect or network effect in a foundation model.」传统软件的护城河来自网络效应（Windows、Google、iOS），但 LLM 没有这种机制，任何有足够资本的玩家都能复制。

- **OpenAI 的应对策略**：用心智份额换硬资产，并"talk your way into a self-fulfilling prophecy"——用叙事制造预期，再让预期自我实现。

- **"即兴软件"vs"制度化软件"**：AI 编程让一类此前无法实现的软件变得可能，但"everyone will build their own software"这个预测是完全错误的；将会有更多软件存在，而不是更少。
