## 今日要点

**1. Anthropic Claude 1M context 窗口全面开放，且无长 context 溢价**

Simon Willison 的博客揭示了一个重要定价信号：Opus 4.6 和 Sonnet 4.6 的 100 万 token context 窗口现已正式对所有用户开放，且不收取任何长 context 附加费。这与 OpenAI（GPT-5.4 超过 272,000 tokens 涨价）和 Google（Gemini 3.1 Pro 超过 200,000 tokens 涨价）的定价策略形成直接对比。这是一个明确的竞争信号：Anthropic 正在用定价换市场份额，对于需要处理大量文档的企业和开发者来说是实质性利好。

**2. Shopify CEO 用 autoresearch 模式让 Liquid 模板引擎性能提升 53%**

Simon Willison 详细记录了 Shopify CEO Tobias Lütke 亲自向 Liquid 提交 PR 的事件：借助 Andrej Karpathy 的 autoresearch 概念（让 coding agent 自主跑数百次实验），通过约 120 次自动化实验找到了解析速度提升 53%、内存分配减少 61% 的优化方案。这个案例的深层意义在于：有了 robust 测试套件 + benchmarking 脚本，"让代码变快"就变成了 agent 可以自主执行的目标。高管层级人员重新参与 coding 的趋势正在加速。

**3. Next.js 16.2 发布，定位为"agent-native 框架"**

Vercel CEO Guillermo Rauch 宣布 Next.js 16.2 引入 AGENTS.md + bundled docs 机制，让 coding agent 能够精通当前项目所用的确切版本。同时推出 @vercel/next-browser 工具，让 agent 能够 debug 和优化前端。Rauch 声称在 Vercel 内部生产环境中，agent 发现了专业前端工程师手动遗漏的优化点。这标志着框架本身开始为 agent 使用而设计，是基础设施层面的重要转变。

**4. Box CEO Aaron Levie 对 AI agent 技术栈迭代速度的深度判断**

Levie 的观察极具实操价值：6 个月前花 6 个月打磨的 agent 架构，现在可能已经过时，推倒重来比修补更划算。他具体指出：上下文窗口改善让许多 RAG pipeline 变得不必要，但同样的 tool use 进步又要求你必须支持代码执行沙箱。"一个能力被替代，一个新能力开启"的循环将持续多年——对没有速度和敏捷度的团队来说是真实威胁。

**5. OpenAI 发布 ChatGPT for Excel 并整合金融数据，由 GPT-5.4 驱动**

OpenAI 将 ChatGPT 直接嵌入 Excel，并增加了金融数据 API 集成，定向服务受监管行业（金融、合规环境）的建模、研究和分析需求。这是 OpenAI 企业落地策略的一次具体动作——从通用工具向垂直场景专项产品演进，与 Microsoft 的深度绑定进一步强化。

### GitHub Trending (Daily)

**opendataloader-project/opendataloader-pdf**

- **核心论点**：一个专为 AI/RAG 场景设计的开源 PDF 解析器，在综合基准测试中排名第一（总分 0.90），支持本地运行、无需 GPU，并计划 Q2 2026 推出免费的 PDF 无障碍自动打标签功能。
- **关键要点**：
  - 混合模式（本地 Java 处理 + AI 后端）综合得分 0.90，表格精度 0.93，优于 docling（0.86）、marker（0.83）和 pymupdf4llm（0.57）
  - 本地模式处理速度 0.05 秒/页（20+ 页/秒），混合模式 0.43 秒/页，均无需 GPU
  - 输出带 bounding box 的结构化 JSON，支持 RAG 溯源引用（"点击定位到原文"体验）
  - 内置 prompt injection 防护，自动过滤隐藏文本、可疑不可见层
  - 与 LangChain 有官方集成（langchain-opendataloader-pdf）
  - Q2 2026 将开源 PDF 无障碍自动打标签功能（Apache 2.0），与 PDF Association 和 veraPDF 开发者合作构建，这是开源领域首个端到端 Tagged PDF 生成工具
  - Apache 2.0 许可，可商用，企业级 PDF/UA 导出为付费功能
- **值不值得深读**：做 RAG pipeline 的开发者强烈推荐深读，有完整的 benchmark 数据和代码示例。
- [原文链接](https://github.com/opendataloader-project/opendataloader-pdf)

### Latent Space（YouTube）

**Dreamer: the Agent OS for Everyone — David Singleton**

核心观点：要让 AI agent 真正规模化服务普通消费者（而非技术人员），必须先构建类似操作系统的基础设施层，agent 权限管理如同 OS 的 kernel/user ring 架构，缺一不可。

David Singleton 是 Stripe 前 CTO，现为 Dreamer 的创始人。Dreamer 是一个面向所有人（包括完全没有技术背景的用户）的 agent 发现、构建和使用平台，核心是一个可自定义名字和人格的个人 agent"Sidekick"。

三个值得关注的具体洞见：

第一，Singleton 明确指出"隐私和安全是 agent 规模化的核心瓶颈"——如果用户不能信任 agent，再酷的功能也没用，这是 Dreamer 要先解决的问题。

第二，他们选择将 sidekick 类比为 OS kernel，将 agent 和 app 类比为不同权限的 user，这是对 agent 权限隔离问题的一种架构回答——不同 agent 之间不能随意共享数据。

第三，他提到"vibe-coded apps 会随意抓取你所有数据、互相之间无法协作"，这正是需要基础 OS 层的原因——没有这层，消费级 agent 生态就会是一团乱麻。

原话："if you try to pick off just one piece of this, you can't actually make it work for people at scale."
